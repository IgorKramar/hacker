# Внедрение команд (Command Injection)

## Описание
Внедрение команд — уязвимость, позволяющая злоумышленнику выполнять произвольные команды в операционной системе хоста. Возникает когда приложение передает пользовательский ввод в системные команды без должной валидации и экранирования.

## Уровень
Продвинутый

## Примеры использования

### Выполнение произвольных команд
- Атака позволяет выполнять произвольные команды в операционной системе хоста.
- Пример использования команды `uname` для получения системной информации.
- Возможность изменения текущей работы системы или получения паролей.

### Процесс атаки
- Перехват запроса с пользовательским вводом.
- Вставка системных команд в пользовательский ввод.
- Использование специальных символов для разделения команд (например, `;`, `|`, `&&`).
- Выполнение команд на сервере через уязвимое приложение.

### Примеры команд
- Получение системной информации: `uname -a`
- Чтение файлов: `cat /etc/passwd`
- Выполнение произвольных команд для получения доступа к системе.

## Типы уязвимостей

### Прямое выполнение команд
- Приложение напрямую передает пользовательский ввод в системные команды.
- Отсутствие валидации и экранирования.
- Использование функций типа `system()`, `exec()`, `shell_exec()`.

### Небезопасная обработка ввода
- Отсутствие санитизации пользовательского ввода.
- Неправильное экранирование специальных символов.
- Использование небезопасных функций для выполнения команд.

## Защита от внедрения команд
- Избегание выполнения системных команд с пользовательским вводом.
- Использование параметризованных вызовов вместо конкатенации строк.
- Валидация и санитизация всего пользовательского ввода.
- Использование whitelist для разрешенных команд и параметров.
- Применение принципа наименьших привилегий для выполнения команд.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных приложениях или с явного разрешения.
- Понимание юридических последствий незаконного использования.
