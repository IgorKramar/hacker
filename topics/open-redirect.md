# Уязвимости открытого перенаправления (Open Redirect)

## Описание
Уязвимости открытого перенаправления — уязвимость, возникающая когда приложение небезопасно вводит данные, контролируемые пользователем, в цель перенаправления. Злоумышленник может создать URL-адрес, который вызывает перенаправление на произвольный внешний домен, что может стать началом фишинговой атаки.

## Уровень
Средний

## Примеры использования

### Небезопасное перенаправление
- Приложение небезопасно вводит данные, контролируемые пользователем, в цель перенаправления.
- Злоумышленник может создать URL-адрес, который вызывает перенаправление на произвольный внешний домен.
- Это может стать началом фишинговой атаки.

### Пример уязвимости
- При нажатии на кнопку выхода из системы появляется URL веб-сайта.
- Изменение этого URL позволяет перенаправить пользователя на другой сайт.
- Пример: изменение URL на Google.com приводит к перенаправлению на этот сайт.

### Процесс атаки
- Обнаружение параметра перенаправления в URL (например, `redirect=`, `return=`, `url=`).
- Изменение значения параметра на внешний домен.
- Создание фишинговой ссылки, которая выглядит как легитимная.
- Перенаправление пользователя на вредоносный сайт.

## Типы уязвимостей

### Прямое использование пользовательского ввода
- Приложение напрямую использует параметр URL для перенаправления.
- Отсутствие валидации домена или URL.
- Разрешение перенаправления на любые внешние домены.

### Отсутствие whitelist
- Приложение не проверяет разрешенные домены для перенаправления.
- Отсутствие ограничений на внешние URL.
- Позволяет перенаправлять на любые домены.

## Защита от открытого перенаправления
- Использование whitelist для разрешенных URL перенаправления.
- Валидация доменов перед перенаправлением.
- Ограничение перенаправления только внутренними URL.
- Использование относительных путей вместо абсолютных URL.
- Проверка подлинности URL перед перенаправлением.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных приложениях или с явного разрешения.
- Понимание юридических последствий незаконного использования.
