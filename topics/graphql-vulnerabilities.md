# Уязвимости GraphQL

## Описание
GraphQL — язык запросов к данным с открытым исходным кодом для API, который многие веб-сайты используют для упрощения доступа к данным. GraphQL может иметь уязвимости, связанные с отсутствием ограничений на запросы, раскрытием информации и возможностью выполнения атак методом перебора.

## Уровень
Средний

## Примеры использования

### Особенности GraphQL
- GraphQL — язык запросов к данным с открытым исходным кодом для API.
- Многие веб-сайты используют GraphQL для упрощения доступа к данным.
- Возможность обновления загруженного файла через GraphQL открывает уязвимости для атак.

### Уязвимости GraphQL
- Отсутствие ограничений на сложность запросов.
- Возможность выполнения глубоких вложенных запросов.
- Раскрытие информации через introspection запросы.
- Отсутствие валидации и проверки запросов.

### Атаки методом перебора
- Важность верификации и проверки запросов для предотвращения получения информации методом "грубой силы".
- Возможность выполнения множественных запросов для получения данных.
- Отсутствие rate limiting на GraphQL эндпоинтах.

## Типы уязвимостей

### Раскрытие информации
- Introspection запросы могут раскрыть структуру схемы.
- Получение информации о типах, полях и отношениях.
- Раскрытие внутренней логики приложения.

### Отсутствие ограничений
- Нет ограничений на глубину и сложность запросов.
- Возможность выполнения ресурсоемких запросов.
- Отсутствие rate limiting и валидации.

### Небезопасные мутации
- Возможность обновления данных без должной авторизации.
- Отсутствие проверки прав доступа.
- Манипуляция данными через GraphQL мутации.

## Защита от уязвимостей GraphQL
- Отключение introspection в production окружении.
- Ограничение глубины и сложности запросов.
- Реализация rate limiting для GraphQL эндпоинтов.
- Валидация и проверка всех запросов.
- Правильная авторизация для мутаций и запросов.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных приложениях или с явного разрешения.
- Понимание юридических последствий незаконного использования.
