# Небезопасная десериализация (Insecure Deserialization)

## Описание
Небезопасная десериализация — уязвимость, возникающая когда данные, контролируемые пользователем, десериализуются веб-сайтом без должной валидации. Злоумышленник может манипулировать сериализованными объектами для передачи вредоносных данных в код приложения, что требует знания внутренних систем и классов приложения.

## Уровень
Продвинутый

## Примеры использования

### Манипуляция сериализованными объектами
- Данные, контролируемые пользователем, десериализуются веб-сайтом.
- Злоумышленник может манипулировать сериализованными объектами для передачи вредоносных данных в код приложения.
- Требует знания внутренних систем и классов приложения.

### Процесс атаки
- Обнаружение места, где приложение десериализует данные.
- Анализ структуры сериализованных объектов.
- Создание вредоносного сериализованного объекта.
- Передача объекта в приложение для выполнения произвольного кода.

### Типы атак
- Выполнение произвольного кода через десериализацию.
- Изменение логики приложения через манипуляцию объектами.
- Получение доступа к конфиденциальным данным.
- Обход аутентификации и авторизации.

## Типы уязвимостей

### Небезопасная десериализация
- Приложение десериализует данные без валидации.
- Отсутствие проверки целостности данных.
- Разрешение десериализации произвольных объектов.

### Отсутствие валидации
- Приложение не проверяет источник и содержимое сериализованных данных.
- Отсутствие ограничений на типы десериализуемых объектов.
- Позволяет выполнять произвольный код.

## Защита от небезопасной десериализации
- Избегание десериализации данных, контролируемых пользователем.
- Использование безопасных форматов данных (JSON вместо native serialization).
- Валидация и проверка целостности данных перед десериализацией.
- Использование whitelist для разрешенных типов объектов.
- Изоляция процесса десериализации с минимальными привилегиями.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных приложениях или с явного разрешения.
- Понимание юридических последствий незаконного использования.
