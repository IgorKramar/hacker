# Массовое задание (Mass Assignment)

## Описание
Массовое задание (Mass Assignment) — уязвимость, возникающая когда приложение позволяет пользователю устанавливать значения полей, которые не должны быть доступны напрямую. Злоумышленник может добавить дополнительные поля в запрос (например, "администратор") и получить привилегированный доступ.

## Уровень
Средний

## Примеры использования

### Перехват HTTP-запросов
- Использование инструментов для перехвата HTTP-запросов (Perpsuit Community Edition).
- Анализ формы для входа в систему и отправка запроса на API.
- Добавление поля "администратор" в запрос для создания учётной записи администратора.

### Процесс атаки
- Перехват запроса регистрации или обновления пользователя.
- Анализ структуры запроса и доступных полей.
- Добавление привилегированных полей (например, `admin: true`, `role: "administrator"`).
- Отправка модифицированного запроса для получения повышенных привилегий.

## Типы уязвимостей

### Небезопасная десериализация объектов
- Приложение автоматически присваивает значения полям из запроса.
- Отсутствие валидации и фильтрации полей.
- Позволяет устанавливать значения для внутренних полей.

## Защита от Mass Assignment
- Использование whitelist для разрешенных полей.
- Явное указание полей, которые могут быть установлены пользователем.
- Валидация и фильтрация всех входящих данных.
- Использование DTO (Data Transfer Objects) для ограничения полей.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных приложениях или с явного разрешения.
- Понимание юридических последствий незаконного использования.
