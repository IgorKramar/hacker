# Атака с обходом пути (Path Traversal)

## Описание
Атака с обходом пути (Path Traversal) — уязвимость, возникающая когда приложение позволяет манипулировать переменными, ссылающимися на файлы, для получения доступа к файлам за пределами корневой папки веб-сайта. Злоумышленник может получить доступ к конфиденциальным файлам на сервере.

## Уровень
Продвинутый

## Примеры использования

### Манипуляция путями к файлам
- Манипулирование переменными, ссылающимися на файлы, позволяет получить доступ к файлам за пределами корневой папки веб-сайта.
- Пример: загрузка файла с изменённым именем для доступа к файлам паролей.
- Необходимость перебора всех возможных комбинаций для достижения цели.

### Процесс атаки
- Обнаружение параметра, который ссылается на файл (например, `file=`, `path=`, `include=`).
- Использование специальных последовательностей для обхода пути (`../`, `..\\`).
- Доступ к файлам за пределами корневой директории веб-сайта.
- Получение доступа к конфиденциальным файлам (пароли, конфигурации, исходный код).

### Примеры обхода
- Использование `../` для перехода в родительские директории.
- Комбинации типа `....//` для обхода фильтров.
- Использование URL-кодирования для обхода валидации.
- Доступ к системным файлам через обход пути.

## Типы уязвимостей

### Небезопасная обработка путей
- Приложение напрямую использует пользовательский ввод для доступа к файлам.
- Отсутствие валидации и санитизации путей.
- Разрешение доступа к файлам вне корневой директории.

### Отсутствие ограничений
- Приложение не ограничивает доступ к определенным директориям.
- Отсутствие проверки пути перед доступом к файлу.
- Позволяет читать любые файлы на сервере.

## Защита от обхода пути
- Валидация и санитизация всех путей к файлам.
- Использование whitelist для разрешенных файлов.
- Ограничение доступа только к определенным директориям.
- Использование абсолютных путей с проверкой.
- Удаление специальных символов из путей.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных приложениях или с явного разрешения.
- Понимание юридических последствий незаконного использования.
