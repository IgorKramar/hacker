# Повышение привилегий через WSL (Windows Subsystem for Linux)

## Описание
Windows Subsystem for Linux (WSL) — подсистема Windows, позволяющая запускать Linux поверх Windows. Неправильная конфигурация или уязвимости в WSL могут привести к повышению привилегий и получению доступа к системе Windows с правами администратора или SYSTEM.

## Уровень
Продвинутый

## Примеры использования

### Использование WSL для доступа
- Поиск файлов "wsl.exe" и "bash.exe" в системе.
- Запуск "wsl.exe" для получения доступа к Linux-терминалу.
- Использование Python для создания оболочки tty через `import pty` и `spawn`.

### Процесс эксплуатации
1. Получение начальной точки опоры на системе Windows.
2. Обнаружение установленного WSL.
3. Запуск WSL для получения доступа к Linux-терминалу.
4. Использование Linux-инструментов для эскалации привилегий.
5. Получение доступа к системе Windows с повышенными привилегиями.

### Использование Python
- Запуск "python" и "import pty" для создания оболочки tty.
- Использование "spawn" для запуска оболочки и получения доступа к файловой системе.
- Создание интерактивной оболочки через WSL.

## Уязвимости WSL

### Неправильная конфигурация
- WSL может быть неправильно настроен, что приводит к уязвимостям.
- Отсутствие ограничений на доступ к файловой системе Windows.

### Интеграция с Windows
- WSL имеет доступ к файловой системе Windows.
- Может быть использован для доступа к защищённым ресурсам Windows.

## Защита от WSL уязвимостей
- Правильная настройка WSL с ограничениями доступа.
- Регулярное обновление WSL и системы Windows.
- Мониторинг использования WSL на системе.
- Ограничение доступа к WSL для обычных пользователей.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных системах или с явного разрешения.
- Понимание юридических последствий незаконного использования.
