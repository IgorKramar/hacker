# Раскрытие информации (Information Disclosure)

## Описание
Раскрытие информации — уязвимость, возникающая когда приложение непреднамеренно раскрывает конфиденциальную информацию о технологическом стеке, инфраструктуре сервера, структуре данных или внутренней логике. Эта информация может помочь злоумышленникам в дальнейших атаках.

## Уровень
Средний

## Примеры использования

### Раскрытие через синтаксические ошибки
- Пример раскрытия информации через синтаксические ошибки JSON.
- Получение информации о технологическом стеке из сообщений об ошибках.
- Анализ структуры данных через детальные сообщения об ошибках.

### Раскрытие информации о сервере
- Получение информации о технологическом стеке и инфраструктуре сервера.
- Версии используемого программного обеспечения.
- Информация об операционной системе и конфигурации.

### Раскрытие конфиденциальной информации
- Запрос GET в API пользователя выдаёт информацию о паролях.
- Раскрытие хэшей паролей через API.
- Знание хэша пароля позволяет выполнить атаку методом перебора.
- Пример раскрытия конфиденциальных данных, которого не должно происходить.

### Раскрытие скрытой функциональности API
- Анализ документации API и поиск скрытых функций.
- Отправка различных запросов для выявления уязвимостей.
- Использование инструментов (Skipfish) для охвата всех возможных слов в API.
- Обнаружение недокументированных эндпоинтов и функций.

## Источники раскрытия информации

### Сообщения об ошибках
- Детальные сообщения об ошибках с информацией о стеке.
- Stack traces с информацией о коде.
- Информация о структуре базы данных.

### Заголовки HTTP
- Версии серверов в заголовках.
- Информация о технологиях в заголовках.
- Конфигурационная информация.

### API ответы
- Избыточная информация в ответах API.
- Раскрытие внутренних структур данных.
- Информация о пользователях и системе.

## Защита от раскрытия информации
- Унификация сообщений об ошибках для пользователей.
- Удаление детальной информации из production окружения.
- Ограничение информации в заголовках HTTP.
- Валидация и фильтрация данных в ответах API.
- Использование общих сообщений об ошибках.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных приложениях или с явного разрешения.
- Понимание юридических последствий незаконного использования.

- Best practices для безопасной обработки ошибок
