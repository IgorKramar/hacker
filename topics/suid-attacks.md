# SUID-атаки (SUID Attacks)

## Описание
SUID (Set User ID) — специальный бит разрешений в Linux, позволяющий исполняемому файлу запускаться с правами владельца файла, а не пользователя, который его запустил. Злоупотребление SUID-битами может привести к эскалации привилегий до root.

## Уровень
Продвинутый

## Поиск файлов с SUID

### Ручной поиск
- Использование команды `find` для поиска файлов с разрешениями SUID.
- Указание параметров поиска: начало с корневого каталога, поиск файлов, принадлежащих пользователю root.
- Направление вывода в `/dev/null` для дальнейшего анализа.

### Использование инструментов
- Использование GitHub.io для поиска SUID.
- Сравнение и сопоставление информации о sudo паролях и SUID.
- Упоминание о возможных уязвимостях SUID.

## Анализ разрешений

### Понимание SUID
- Объяснение настроек разрешений и групп доступа.
- Пример использования `ls -la` для просмотра прав доступа к файлу.
- Объяснение работы разрешений suid и их влияния на привилегии.

### Уязвимые бинарники
- Поиск знакомых sudo и паролей, но также поиск новых уязвимостей.
- Использование GTFO Bins для поиска уязвимых бинарников.

## Примеры эксплуатации

### Systemctl
- Systemctl работает с битом SUID и может использоваться для доступа к файловой системе с повышенными привилегиями.
- Создание переменной окружения и временного сервиса systemctl или systemd.
- Выполнение команды `/bin/sh` для запуска процесса без идентификатора.
- Установка переменной окружения target и настройка ссылки для запуска файла через systemctl.
- Создание и включение ссылки для выполнения необходимых действий.

### Vim
- Использование vim с SUID для получения root-доступа.
- Создание GTFO-контейнеров для дальнейшего анализа.

### Bash
- Альтернативные способы получения root-доступа через bash с SUID.

## Процесс эксплуатации
1. Поиск файлов с SUID битом.
2. Анализ уязвимых бинарников.

4. Создание и выполнение эксплойта для получения root.

## Защита от SUID-атак
- Минимизация использования SUID битов.
- Регулярный аудит файлов с SUID.
- Ограничение доступа к уязвимым бинарникам.
- Применение принципа наименьших привилегий.
