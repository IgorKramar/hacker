# Основы работы компьютера

## Введение

Компьютер — это программируемое электронное устройство, способное хранить, обрабатывать и передавать данные. Понимание фундаментальных принципов его работы важно для всех, кто взаимодействует с цифровой техникой: это основа для освоения программирования, понимания архитектуры устройств и грамотного использования вычислительных ресурсов.

## Ключевые компоненты компьютера

**1. Центральный процессор (CPU)**

ЦП (центральный процессор) — "мозг" компьютера, который анализирует и выполняет инструкции программ. В современных компьютерах процессор состоит из одного или нескольких ядер, каждое из которых может параллельно выполнять вычисления. Процессор содержит регистры для временного хранения данных, арифметико-логическое устройство (ALU) для операций и контрольное устройство (CU) для управления порядком выполнения команд.

**Архитектура фон Неймана** — классическая модель, в которой память и процессор разделены, а данные и инструкции хранятся в единой памяти[84]. Процессор извлекает инструкции по циклу: fetch (выборка), decode (декодирование), execute (выполнение)[83][93].

**2. Оперативная память (RAM)**

RAM (Random Access Memory) — область хранения данных, с которой процессор работает напрямую. DRAM (динамическая RAM) — основной тип ОЗУ, нужна для хранения работающих программ и временных данных. SRAM — более быстрая статическая память, используется для кэш-памяти процессора[77][128].

**3. Постоянная память (ROM, SSD, HDD)**

ROM — память только для чтения, содержит базовую программу запуска (BIOS/UEFI)[71]. SSD/HDD — основные устройства долговременного хранения пользовательских данных и программ.

**4. Видеопамять (VRAM), кэш, регистры**

Видеокарта содержит собственную память (VRAM) для быстрой обработки графики. Кэш — сверхбыстрая память между ядром CPU и RAM (уровни L1, L2, L3)[72]. Регистры — быстрейшая область хранения внутри CPU для текущих операций.

## Основные единицы данных

- **Бит** — минимальная единица информации, может быть 0 или 1[68].
- **Байт** — 8 бит, стандартная единица хранения одного символа, например, буквы или числа[62][68].
- **Килобайт, мегабайт, гигабайт** — производные единицы для измерения объема данных.

Данные в компьютере представлены в двоичной системе счисления, где каждая единица памяти — это бит, объединенный в байты и далее в большие группы.

## Работа процессора: цикл выполнения инструкций

Процессор выполняет программы по циклу, который называют **Fetch–Decode–Execute (FDE)**:

1. **Fetch** — процессор получает команду из памяти по адресу, определенному счётчиком команд (Program Counter, PC).
2. **Decode** — декодирует команду, определяя операцию (например, сложение, перемещение данных).
3. **Execute** — выполняет команду, изменяя внутренние данные, память или отправляя инструкции устройствам.

Пример простого цикла на ассемблере (x86):

```assembly
MOV AX, 5       ; Помещаем число 5 в регистр AX
ADD AX, 2       ; Прибавляем к AX число 2
MOV BX, AX      ; Переносим результат в регистр BX
```

Рассмотрим простую операцию:
- CPU получает инструкцию из оперативной памяти (fetch)
- Декодирует: "сложить 5 и 2" (decode)
- Исполняет: складывает и записывает в регистр (execute)

## Работа с памятью

- **RAM** быстро доступна, используется для временного хранения данных и инструкций.
- **ROM** используется для хранения программы начального запуска — BIOS, и не стирается при выключении питания[71].
- **Кэш**: L1 — самый быстрый, находится в ядре CPU; L2, L3 — между ядром и RAM; ускоряет доступ к часто используемым данным[72].
- **SRAM vs DRAM**:
    - SRAM быстрее (используется для кэша), не требует обновления, но дороже и имеет меньшую ёмкость[128].
    - DRAM дешевле и обеспечивает большую ёмкость, но требует периодического обновления и медленнее.

## Системы счисления

- **Двоичная (бинарная) система (base-2)** — используется в низкоуровневых операциях памяти и инструкций.
- **Шестнадцатеричная система (base-16)** — удобна для представления больших бинарных чисел в компактной форме; часто встречается в ассемблере и адресации памяти.

Пример преобразования числа:
- Двоичное `1101110,11₂` → Шестнадцатеричное `6E,C₁₆`.

## Прерывания и устройства ввода-вывода

Процессор может быть "прерван" событиями, такими как ввод с клавиатуры или завершение операции с диском. Обработчиком прерывания занимается специальная часть ОС или прошивки, которая реагирует на события с внешних устройств.

## Примеры кода

### Проверка флага нуля (Zero Flag) на x86-ассемблере

```assembly
MOV AX, 2      ; загрузить 2 в регистр AX
SUB AX, 2      ; вычесть 2, AX = 0
JZ ZeroFlagSet ; переход, если установлен флаг нуля
```

### Работа с битами в Python

```python
# Выставить третий бит в числе
num = 0b0010
num = num | (1 << 2)  # 0b0110
# Проверить установлен ли бит
print(bool(num & (1 << 2)))  # True
```
