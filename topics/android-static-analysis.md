# Статический анализ Android приложений

## Описание
Статический анализ Android приложений — метод тестирования безопасности, который проверяет код и настройки приложения без его запуска. Позволяет выявить уязвимости в коде, конфигурации и зависимостях до развертывания приложения.

## Уровень
Средний

## Процесс статического анализа

### Декомпиляция APK
- Использование dex2jar для преобразования DEX файлов в JAR.
- Использование Jadx для декомпиляции APK в читаемый Java код.
- Извлечение исходного кода из скомпилированного приложения.
- Анализ структуры приложения и его компонентов.

### Проверка криптографии
- Отсутствие слабого шифрования в коде приложения.
- Проверка на наличие жёстко закодированных ключей шифрования.
- Анализ использования криптографических библиотек.
- Проверка правильности реализации криптографических алгоритмов.

### Обфускация кода
- Проверка на наличие обфусцированного кода.
- Анализ методов обфускации и их эффективности.
- Выявление скрытых функций и логики приложения.

### Использование API
- Проверка на отсутствие небезопасных API.
- Анализ использования системных API.
- Проверка правильности обработки данных через API.

### Конфиденциальные данные
- Отсутствие жёстко запрограммированных паролей в коде.
- Проверка на наличие токенов и ключей API в коде.
- Анализ хранения конфиденциальных данных.
- Проверка правильности обработки персональных данных.

### Внешние библиотеки
- Проверка на наличие уязвимостей в сторонних библиотеках.
- Анализ зависимостей приложения.
- Проверка версий используемых библиотек на известные уязвимости.
- Оценка безопасности внешних компонентов.

### Файл манифеста
- Проверка на наличие опасных разрешений (permissions).
- Анализ компонентов приложения (activities, services, receivers).
- Проверка конфигурации безопасности приложения.
- Анализ экспортируемых компонентов.

## Инструменты для статического анализа

### Декомпиляция
- **dex2jar**: преобразование DEX файлов в JAR для анализа.
- **Jadx**: декомпиляция APK в читаемый Java код с графическим интерфейсом.

### Анализ кода
- Ручной анализ декомпилированного кода.
- Поиск уязвимостей в логике приложения.
- Анализ обработки пользовательского ввода.

## Типы выявляемых уязвимостей

### Уязвимости безопасности
- Небезопасное хранение данных.
- Неправильная обработка пользовательского ввода.
- Утечки конфиденциальной информации.
- Небезопасные криптографические реализации.

### Проблемы конфигурации
- Опасные разрешения в манифесте.
- Экспортируемые компоненты без проверки прав.
- Неправильная настройка безопасности.

## Защита от статического анализа

### Обфускация
- Использование инструментов обфускации кода.
- Затруднение чтения и понимания кода.
- Защита от обратной инженерии.

### Шифрование
- Шифрование конфиденциальных данных.
- Использование безопасных методов хранения ключей.
- Избегание жёстко закодированных секретов.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только собственных приложений или с явного разрешения.
- Понимание юридических последствий незаконного использования.
