# SMS-спуфинг (SMS Spoofing)

## Описание
SMS-спуфинг — техника подделки SMS-сообщений с любым именем отправителя для выполнения различных сценариев, включая фишинг, социальную инженерию и обход систем безопасности. Позволяет отправлять сообщения от имени известных организаций или знакомых людей.

## Уровень
Продвинутый

## Примеры использования

### Подмена отправителя
- Подделка SMS-сообщений с любым именем отправителя.
- Отправка сообщений от имени известных организаций (Google, банки).
- Отправка сообщений от имени знакомых людей.
- Простота подмены ID отправителя через SMS-шлюзы.

### Сценарии использования
- Отправка сообщения от имени Google для восстановления учётной записи.
- Загрузка приложений или переход по ссылкам через поддельные SMS.
- Фишинговые атаки через SMS-сообщения.
- Социальная инженерия для получения конфиденциальной информации.

### Фишинговые сообщения
- Сообщения, выглядящие как отправленные с известных сервисов (Gmail).
- Уведомления о взломе и блокировке учётной записи.
- Просьбы перейти по ссылкам для восстановления доступа.
- Ссылки на поддельные страницы входа с включённым HTTPS.

## Инструменты и сервисы

### SMS-шлюзы
- **Octopush**: простой вариант для подмены SMS без сложных требований.
- **Twilio**: сложность работы из-за требований конфиденциальных документов.
- Другие SMS-шлюзы для подмены отправителя.

### Процесс отправки
- Регистрация на SMS-шлюзе (email, пароль, номер телефона).
- Пополнение счёта и ожидание поступления кредита.
- Создание SMS-кампании с указанием имени отправителя.
- Ввод номера телефона получателя и отправка сообщения.

## Важность соответствия
- Сообщение должно соответствовать личности отправителя.
- Правдоподобность поддельных SMS для успешной атаки.
- Использование воображения для реализации различных сценариев.

## Защита от SMS-спуфинга
- Проверка подлинности SMS-сообщений через официальные каналы.
- Осторожность при переходе по ссылкам из SMS.
- Проверка подлинности отправителя перед предоставлением информации.
- Использование двухфакторной аутентификации через приложения, а не SMS.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных номерах или с явного разрешения.
- Понимание юридических последствий незаконного использования.
- SMS-спуфинг против реальных людей является незаконным.
