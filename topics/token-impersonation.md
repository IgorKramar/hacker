# Олицетворение токенов (Token Impersonation)

## Описание
Олицетворение токенов — техника использования токенов доступа других пользователей или процессов для получения доступа к ресурсам, доступным только для администраторов или SYSTEM. Позволяет выполнять действия от имени привилегированных пользователей без знания их паролей.

## Уровень
Продвинутый

## Примеры использования

### Использование токенов для доступа к ресурсам
- Использование олицетворения токенов для получения доступа к ресурсам, доступным только для администраторов.
- Метод может быть полезен для выполнения различных задач, таких как сброс хэшей с контроллера домена.
- Выполнение действий от имени привилегированных пользователей.

### Использование Metasploit
- Встроенный инструмент "get system" в Metasploit позволяет улучшить систему несколькими способами.
- Включает олицетворение и другие привилегии.
- Важно понимать эти привилегии с самого начала, чтобы не навредить системе.

### Процесс олицетворения
1. Получение токена доступа привилегированного пользователя или процесса.
2. Олицетворение токена для выполнения действий от его имени.
3. Использование привилегий токена для доступа к защищённым ресурсам.

## Типы токенов

### Токены доступа пользователей
- Токены, связанные с пользовательскими сессиями.
- Могут содержать привилегии пользователя.

### Токены процессов
- Токены, связанные с запущенными процессами.
- Могут содержать привилегии процесса, включая SYSTEM.

## Защита от олицетворения токенов
- Ограничение доступа к токенам доступа.
- Мониторинг подозрительной активности с токенами.
- Применение принципа наименьших привилегий.
- Регулярный аудит использования токенов.

## Важные предупреждения
- Использование только в образовательных целях и с разрешения.
- Тестирование только на собственных системах или с явного разрешения.
- Понимание юридических последствий незаконного использования.
