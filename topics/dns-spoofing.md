# Подмена DNS (DNS Spoofing)

## Описание
Подмена DNS-сервера для перенаправления запросов на поддельные сайты, позволяющая хакеру контролировать браузер жертвы и выполнять различные атаки (фишинг, кража данных).

## Уровень
Продвинутый

## Примеры использования
- Подделка DNS-сервера для перенаправления запросов.
- Перенаправление жертвы на поддельные сайты вместо настоящих.
- Контроль браузера жертвы через подмену DNS.
- Выполнение фишинговых атак через подмену DNS.
- Использование в комбинации с Evil Twin атаками.
- Использование Bettercap для автоматизации DNS spoofing атак.
- Перенаправление пользователей на вредоносные веб-сайты через подмену DNS-запросов.
- Использование веб-серверов (Apache) для создания вредоносных сайтов.

## Защита от подмены DNS
- Использование VPN для шифрования трафика и защиты DNS-запросов.
- Использование DNS-over-HTTPS (DoH) или DNS-over-TLS (DoT).
- Проверка сертификатов при подключении к сайтам.
- Использование надежных DNS-серверов (например, Cloudflare, Google DNS).

## Ресурсы для изучения
- Методики защиты от подмены DNS
- DNS-over-HTTPS и DNS-over-TLS
- Best practices для безопасного использования DNS
