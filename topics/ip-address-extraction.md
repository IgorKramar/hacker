# Извлечение IP-адресов

## Описание
Извлечение и сканирование IP-адресов для обнаружения активных устройств в локальных и общедоступных сетях. Включает определение типов IP-адресов, сканирование портов и анализ результатов для выявления уязвимостей.

## Уровень
Средний

## Типы IP-адресов

### Локальные IP-адреса
- Принадлежат устройствам внутри сети (телефон, ноутбук, смарт-телевизор).
- Используются для внутренней коммуникации в локальной сети.
- Автоматически определяются инструментами сканирования.

### Общедоступные IP-адреса
- Принадлежат маршрутизатору и доступны из любой точки мира.
- Могут быть просканированы из интернета.
- Используются для доступа к устройствам извне локальной сети.

## Примеры использования

### Сканирование локальной сети
- Автоматическое определение диапазона IP-адресов сети.
- Обнаружение активных устройств в локальной сети.
- Определение статуса устройств по цветовой индикации:
  - Зелёный — устройство реагирует на пинг.
  - Красный — устройство не реагирует.
  - Синий — устройство подключено, но без открытых портов.

### Работа с портами
- Сканирование популярных портов: 21 (FTP), 22 (SSH), 23 (Telnet), 80 (HTTP), 445 (SMB), 3389 (RDP), 5900 (VNC).
- Отображение только устройств с открытыми портами.
- Открытие IP-адресов в веб-браузере для доступа к страницам входа в систему маршрутизатора.

### Сканирование общедоступных IP-адресов
- Сканирование общедоступных IP-адресов из интернета.
- Использование сайтов для выбора диапазона IP-адресов (например, nirsoft.net/countryip).
- Анализ результатов сканирования для обнаружения открытых портов.
- Сканирование может занять много времени, особенно для больших диапазонов.

## Анализ результатов
- Открытые порты, такие как FTP и SSH, могут быть использованы для взлома.
- Результаты сканирования можно открыть в веб-браузере для просмотра страниц входа.
- Определение уязвимых устройств по открытым портам.

## Безопасность

### Риски открытых портов
- Открытые порты SSH, Telnet или FTP представляют риск для безопасности.
- Рекомендуется закрыть все ненужные порты на маршрутизаторе.
- Использование VPN для сокрытия следов при сканировании.

### Защита от сканирования
- Закрытие всех ненужных портов на маршрутизаторе.
- Использование файрволов для блокировки нежелательного трафика.
- Регулярный мониторинг сетевой активности.

## Ресурсы для изучения

- Методики безопасного сканирования сетей
